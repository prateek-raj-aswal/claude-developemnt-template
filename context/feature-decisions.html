Feature Decisions & Future Ideas

Persistent record of every feature decision made during planning sessions — both what was selected for implementation and what was deferred or rejected. Agents read this file to avoid re-litigating decided questions. Updated by clarify during PLAN mode and by doc-writer after each story completes.

Status key: PLANNED = in the current backlog (has story IDs)  |  IN_PROGRESS = story actively being built  |  DONE = story completed and shipped  |  DEFERRED = recognised, not in current scope  |  REJECTED = explicitly ruled out with a reason

1. Decisions In Scope (Planned / In Progress / Done)

Features and capabilities that have been decided and assigned to user stories. The clarify agent appends rows here during every PLAN invocation. Rows are never removed — update status as work progresses.

ID Feature / Capability Decision Made Story IDs Status ADR Notes
FD-001 e.g. User Authentication (login, signup, forgot-password, reset-password) 2026-06-04 US-001, US-002, US-003, US-004 PLANNED ADR-001 Full auth journey — includes edit-profile and account deletion

2. Deferred Ideas (Future Scope)

Features that were raised during planning but explicitly deferred to a future phase. The clarify agent adds rows here when a feature is mentioned but excluded from the current backlog. Preserving them prevents re-discovery work in future planning sessions.

ID Feature / Idea Raised On Why Deferred Target Phase / Milestone Status
FI-001 e.g. OAuth2 social login (Google, GitHub) 2026-06-04 v1 scope is email/password only — social login adds complexity Phase 2 / post-MVP DEFERRED

3. Rejected Ideas

Features that were explicitly ruled out. Reasons are recorded so agents and humans don't re-propose them without new justification.

ID Feature / Idea Rejected On Reason Status
FR-001 e.g. Mobile app (iOS/Android) for v1 2026-06-04 Web-only for v1 — mobile deferred until web is validated REJECTED

4. Architectural Decisions

Key architectural choices made during design. Each row that says "Yes" under ADR has a corresponding file in docs/adr/. The architect and doc-writer agents maintain this table.

ID Decision Made On Choice Rationale ADR Status
AD-001 e.g. Database engine 2026-06-04 PostgreSQL 16 Strong JSON support, row-level security, team familiarity ADR-001 DONE
